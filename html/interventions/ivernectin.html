Ivermectin¶

Contains the ivermectin_cfg dictionary defining the default of an Ivermectin intervention along with a function to create an ivermectin campaign event.

dtk.interventions.ivermectin.add_ivermectin(config_builder, drug_code, coverage, start_days, trigger_condition_list=[], listening_duration=-1)[source]¶

Add an ivermectin event to the config_builder passed.

Parameters:
  • config_builder – The config builder getting the intervention event
  • drug_code – Can be ‘DAY’, ‘WEEK’ or ‘MONTH’ and drive the Killing_config (see Killing_Config in Ivermectin for more info).
  • coverage – Set the Demographic_Coverage
  • start_days – list of days when to start the ivermectin distribution
  • trigger_condition_list – ivermectin will be distributed when it hears the trigger string event, please note the start_days then is used to distribute the NodeLevelHealthTriggeredIV
  • listening_duration – for how long the NLHTIV will listen for the trigger
Returns:

Nothing
dtk.interventions.ivermectin.ivermectin_config_by_duration(drug_code=None)[source]¶

Returns the correct Killing_Config parameter depending on the drug_code

Parameters:drug_code –

Can be 'DAY', 'WEEK' or 'MONTH' or a number of days and drive the Killing_config (see Killing_Config in Ivermectin for more info).
Returns:a dictionary with the correct Killing_Config / Box_Duration set.
